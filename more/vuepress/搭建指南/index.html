# VuePress搭建之南

编写人：刘崛雄

创建时间：2022.04.14

最新更新时间：2022.04.19

# 概述

  • VuePress 是尤雨溪（vue.js 框架作者）发布的基于 vue 的静态网站生成器，本质为一个vue 的 spa 应用（单页 Web 应用）。

  • 内置 默认主题 ，以支持 Vue 子项目的文档编写需求。

# 目标

  • VuePress搭建与初步运行
  • 项目结构配置
  • 默认主题配置
  • antd组件引入
  • 二次开发组件引入与示例

# VuePress搭建

# 相关环境

  • 包管理工具npm，安装node.js即可使用
  • vuepress：vuepress@1.8.2
  • 热门主题：vuepress-theme-reco【暂未研究】

# 安装运行

  1. 新建文件夹cac-vuepress

  2. 进入文件夹目录，初始化package.json配置文件

    • 方法一：通过npm init命令，填写相关参数后自动生成；

    • 方法二：直接新建package.json配置文件，填入如下内容：

      {
  "name": "cac-vuepress",
  "version": "1.0.0",
  "description": "前端组件库文档",
  "main": "index.js",
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "ljx",
  "license": "MIT",
  "devDependencies": {
    "vuepress": "^1.9.7"
  }
}

  3. 执行npm install -D vuepress命令，安装VuePress 本地依赖

  4. 新建docs文件夹 ，在其中创建 README.md，并写入标题文本

    • 方法一：执行mkdir docs && echo '# Hello VuePress' > docs/README.md命令自动创建
    • 方法二：手动创建

    此时文件夹结构如下：

  5. 在package.json文件中加入如下启动代码

      "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  },

  6. 执行上一步配置的启动命令：npm run docs:dev，执行后VuePress 会启动一个热重载的开发服务器

此时，已经完成 VuePress 文档的安装运行啦~~~

接下来，进行官方推荐的 目录结构 (opens new window) 配置

# 项目结构

VuePress 遵循 约定优于配置 的原则，推荐的目录结构如下

对于上述目录结构中，使用到的部分进行如下介绍：

  • docs/.vuepress: 用于存放全局的配置、组件、静态资源等。

    • docs/.vuepress/components: 该目录中的 Vue 组件将会被自动注册为全局组件。
    • docs/.vuepress/config.js: 配置文件的入口文件，包括页面导航栏和侧边栏布局等。
    • docs/.vuepress/enhanceApp.js: 客户端应用的增强。

  • docs/README.md：首页配置文档。

  • docs/guide

    • 可存放markdown文件，每一个markdown文件对应一个页面。

    • 以vuepress官网为例，存放【指南】链接下的相关文档；

    • 也可另外新建文件夹存放其他链接相关文档；

# 默认的页面路由

docs 目录默认作为targetDir（根目录），所有文件的相对路径，都是对于 docs 目录而言。

默认页面路由地址如下：

文件的相对路径 页面路由地址
/README.md /
/guide/README.md /guide/

# 导航栏链接

通过 themeConfig.nav 增加导航栏链接，nav的相关配置如下：

module.exports = {
    title: 'CAC-Antd-Docs', //导航栏左侧标题
    themeConfig: {
        logo: '/img/logo.png',  // 此处根目录从public/开始
        nav: [
            { text: '首页', link: '/' },
            { text: '指南', link: '/guide/' },
            { text: '组件库', link: '/cac-components/' },
            {
                text: '语言',
                items: [
                    { text: 'Chinese', link: '/yy/' },
                    { text: 'Japanese', link: '/xx/' }
                ]
            },
            {
                text: '了解更多',
                items: [
                    { text: '分组1', items: [{ text: 'xx', link: 'https://cn.bing.com/' }] },
                    { text: '分组2', items: [{ text: 'yy', link: 'https://cn.bing.com/', target: '_self', rel: '' }] }
                ]
            }
        ],
    }
}

# 侧边栏

侧边栏（Sidebar）需配置 themeConfig.sidebar，配置代码如下所示：

        sidebar: {
            '/guide/': [
                {
                    title: '指南',  //组名
                    collapsable: false, //是否可折叠，默认true
                    children: ['','getting-started'],   //该分组下要显示的文件的目录；''代表README.md文件
                },
                {
                    title: '其他',  //组名
                    collapsable: false, //是否可折叠，默认true
                }
            ]
        }

# antd组件引入

由于 VuePress 是一个标准的 Vue 应用，可通过创建一个 .vuepress/enhanceApp.js 文件来做一些应用级别的配置，当该文件存在的时候，会被导入到应用内部。

  1. 在该文件中，加入如下代码：

    import Antd from 'ant-design-vue'
import 'ant-design-vue/dist/antd.css'

export default ({ Vue }) => {
  Vue.use(Antd)
}

  2. 执行npm install ant-design-vue@1.6.2 --save命令安装antd组件

    此时，再启动项目可能会报如下错误：Cannot find module 'core-js/library/fn/xxx/xxx'

    原因：antd组件中依赖的core-js包和vuepress所依赖的core-js包版本不兼容造成的。

    解决方案 => 在vuepress配置文件config.js中进行如下配置：

    module.exports = {
    chainWebpack: config => {
        config.resolve.alias.set('core-js/library/fn', 'core-js/features')
    }
}

  3. 此时，在.md文件中，粘贴 <a-button type="primary">按钮</a-button>代码，即可出现如下显示效果：

需求：编写文档的时候增加 Vue 代码示例，包括组件代码和效果显示，并提供代码的收起/展开功能

经过查找，获得如下开源代码示例展示插件：

  • vuepress-plugin-demo-container (opens new window)【推荐使用】
  • vuepress-plugin-demo-block (opens new window)
  • vuepress-plugin-code-box (opens new window)

vuepress-plugin-demo-container使用步骤：

  1. 安装 vuepress-plugin-demo-container 组件

    npm i vuepress-plugin-demo-container --save-dev

  2. 配置插件

    module.exports = {
  ...
  plugins: ['demo-container']
  ...
}

  3. 语言配置

    由于插件将根据 Vuepress 中匹配的 lang 字段完成语言切换，需在docs/.vuepress/config.js配置 locales 选项；

    点此查看 Vuepress 的多语言支持 (opens new window)

    module.exports = {
	...
    locales: {
        // 键名是该语言所属的子路径
        // 作为特例，默认语言可以使用 '/' 作为其路径。
        '/': {
            lang: 'zh-CN',
            title: 'CAC-Antd-Docs', //导航栏左侧标题
        },
    }
    ...
}

  4. 使用

    在展示的Markdown文件中，编写如下代码：

```html
<template>
	<a-button type="primary">Primary-按钮</a-button>
	<a-button type="danger">Danger-按钮</a-button>
</template>
```
:::

# 组件注册+示例引入

需求：通过antd进行二次开发，形成企业/个人组件后，需进行示例编写。

# 组件注册

  • 方式一

    • 将组件源码放入.vuepress/components中，改文件夹下的 *.vue 文件将会自动地被注册为全局的异步组件；

    • 这些组件可以任意在 Markdown 文件中使用，但是使用方式跟所在路径有关，具体参考在 Markdown 中使用 Vue (opens new window)；

    例如：有如下demo-1、OtherComponent和Bar三个组件路径：

    .docs
└─ .vuepress
   └─ components
      ├─ demo-1.vue
      ├─ OtherComponent.vue
      └─ Foo
         └─ Bar.vue

    其引入的标签分别为<demo-1/>、<OtherComponent/>和<Foo-Bar>；

    注意：以Bar.vue组件为例，从components为起始路径，每多一层文件夹，在引用时就要使用-进行连接

    因此，若要使用组件原名称，就必须将组件放入components根目录。

  • 方式二

    • 在 enhanceApp.js 中，使用语法Vue.component('组件名', component)进行全局注册；
    import TipsByLjx from './components/test/TipsByLjx.vue'

export default ({ Vue }) => {
  Vue.component('TipsByLjx', TipsByLjx)
}

此处采取**最简便的方式**，将组件直接置于.vuepress/components下。

注意：千万别装@vuepress/plugin-register-components@next这玩意进行注册，装了后vuepress-plugin-demo-container后，默认的全局注册就失效了，不知道为啥。。。
