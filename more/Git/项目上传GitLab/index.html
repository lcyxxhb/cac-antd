# 本地项目上传GitLab

创建人：刘崛雄

创建日期：2022.04.20

最近更新日期：2022.04.20

# 前言

项目代码上传，通常分两类情况：

  • 情况一：项目在本地从零开发，未commit或者pull过任何Git版本,需要将自己本地的项目上传到GitLab新建的项目中。
  • 情况二：项目是从远端clone到本地进行开发的，需要将该项目更新回远端GitLab项目中

本操作针对情况一

# 步骤

# 在GitLab上新建Git仓库

点击创建完成之后，生成空的项目仓库。

至此，GitLab上的步骤告一段落。

# 建立本地Git仓库

在本地项目根目录中执行Gti命令行：

  1. 初始化本地仓库

    git init  //初始化本地仓库

  2. 将本地项目的所有文件添加到暂存区

    小数点 “.” 表示添加文件夹下的所有文件；也可以将 “.” 换成具体的文件名

    git add .   //将本地项目的所有文件添加到暂存区

  3. 将暂存区的文件提交到本地仓库,并添加说明信息

    git commit -m "说明信息"

    首次使用git提交代码，需要先配置项目的user.name 和 user.email

    git config user.name “刘某某”
git config user.email “liuxxx@xx.cn”

    若是配置全局，执行

    git config --global user.name “刘某某”
git config --global user.email “liuxxx@xx.cn”

  4. 查看已配置信息

    //查看仓库级的 config，命令：
git config –local --l
//查看全局级的 config，命令：
git config –global --l

至此，本地项目提交到本地仓库完成。

# 本地仓库代码推送到GitLab远端仓库

本地项目根目录中执行Gti命令行

  1. 建立本地仓库和远端GitLab仓库的关联关系

    • 执行git remote相关命令行：
    git remote add origin  https://gitlab*********************.git
    • 若出现错误，例如：fatal:remote origin already exists等，可执行如下命令清除关联关系
    git remote rm origin
    • 查看建立的关联关系
    git remote -v

  2. 本地仓库上传到 GitLab 远端仓库

    • 获取远程库与本地同步合并
    git pull --rebase origin master

    注意：如果远程库不为空必须做这一步，否则后面的提交会失败；

    原因：GitLab中的 README.md 文件不在本地仓库中，可以通过该命令进行代码合并。

    • 把当前分支 master 推送到远程
    git push origin master

    执行完之后，如果无错误就上传成功了。
